Introduction

ฟังก์ชันคือกลุ่มของคำสั่งที่มีอยู่ภายในโปรแกรมสำหรับปฏิบัติงานใดงานหนึ่งโดยเฉพาะ

 Functions
• ฟังก์ชันคือส่วนของโค้ดที่ใช้ซ้ำได้
• ฟังก์ชันมีชื่อและต้องประกาศ
ชื่อฟังก์ชันนั้นดำเนินการฟังก์ชั่นนั้นๆ
• ฟังก์ชันสามารถรับและส่งคืนข้อมูลได้

FILE INPUT/OUTPUT

 Function naming rules
• เพื่อให้โปรแกรมเก็บข้อมูลระหว่างที่ดำเนินการ ต้องบันทึกข้อมูล
 - ข้อมูลจะถูกบันทึกลงในไฟล์โดย
 ทั่วไปแล้วจะอยู่บนดิสก์คอมพิวเตอร์
 - ข้อมูลที่บันทึกไว้สามารถเรียก
 และใช้งานได้ในภายหลัง
• "Writing data to" : บันทึกข้อมูลลงในไฟล์
• "Output file" : เขียนข้อมูลลงในไฟล์ที่กำหนดไว้
• "Reading data from" : กระบวนการดึงข้อมูลจากไฟล์
• "Input file" : ไฟล์ที่ใช้อ่านข้อมูล

TYPE OF FILES

 โดยทั่วไปไฟล์จะแบ่งเป็น 2 ประเภท
•Text file : มีข้อมูลที่เข้ารหัสเป็นข้อความ
• Binary file : ข้อมูลที่ยังไม่ได้แปลงเป็นข้อความ
 มี 2 วิธีที่จะดึงไฟล์ที่เก็บไว้ออกมา
• Sequential access : อ่านไฟล์ตามลำดับตั้งแต่ต้นจนจบ
• Direct access : สามารถข้ามไปยังข้อมูลนั้นๆ ในไฟล์ได้โดยตรง

FILE

 มี 3 ขั้นตอนเสมอที่ต้องทำเมื่อไฟล์ถูกใช้โดยโปรแกรม
• Open the file : การเปิดไฟล์จะสร้างการเชื่อมต่อระหว่างไฟล์กับโปรแกรม การเปิดไฟล์เอาต์พุตมักจะสร้างไฟล์บนดิสก์และอนุญาตให้โปรแกรมเขียนข้อมูลลงไป การเปิดไฟล์อินพุตทำให้โปรแกรมสามารถอ่านข้อมูลจากไฟล์ได้
• Process the file : ในขั้นตอนนี้ ข้อมูลจะถูกเขียนไปยังไฟล์ (หากเป็นไฟล์เอาต์พุต) หรืออ่านจากไฟล์ (หากเป็นไฟล์อินพุต)
• Close the file : เมื่อโปรแกรมใช้งานไฟล์เสร็จแล้ว ไฟล์จะต้องปิด การปิดไฟล์จะยกเลิกการเชื่อมต่อไฟล์จากโปรแกรม

Processing Record

PROCESSING RECORDS

• Record : ชุดข้อมูลที่ใช้อธิบายถึงข้อมูล 1 รายการ
• Field : ข้อมูลเพียงชิ้นเดียวภายในรายการ
• Write record : เข้าถึงไฟล์ตามลำดับโดยการเขียนฟิลด์ทีละรายการ
• Read record : เข้าไฟล์ถึงตามลำดับโดยการอ่านแต่ละฟิลด์จนบันทึกเสร็จสมบูรณ์

 สิ่งสำคัญเมื่อต้องทำงานกับไฟล์ คือ ต้องสามารถทำสิ่งต่อไปนี้ได้
• Add records : เพิ่มข้อมูล
• Display records : แสดงผลข้อมูล
• Search for a specific record : ค้นหาเฉพาะข้อมูลที่บันทึก
• Modify records : ค้นหาและบันทึกข้อมูล
• Delete records : ลบข้อมูลที่บันทึก

Exceptions

EXCEPTION

• Exception : ข้อผิดพลาดที่เกิดขึ้นขณะโปรแกรมกำลังทำงาน
 - มักจะทำให้โปรแกรมหยุดกะทันหัน
• Traceback : ข้อความที่แสดงเกี่ยวกับข้อผิดพลาด โดยจะให้ข้อมูลเป็นหมายเลขบรรทัดทัดที่เกิดการผิดพลาด
 - ระบุประเภทของข้อผิดพลาดและอธิบายถึงข้อผิดพลาดสั้นๆ
• ใน Python จะเป็นคำสั่ง try/except
  General format:
   try:
     statements
   except exceptionName:
     statements

• Try suite : คำสั่งที่อาจก่อให้เกิดข้อยกเว้น
• Handler :
• ข้อผิดพลาดสามารถป้องกันได้ด้วยการเขียน Code อย่างระมัดระวัง
• ข้อผิดพลาดบางอย่างก็ไม่สามารถหลีกเลี่ยงด้วยการเขียน Code อย่างระมัดระวังได้
• หากเกิดข้อผิดพลาดในคำสั่ง Try
 - ข้อผิดพลาดที่ระบุไว้ใน Except
  - คำสั่งที่ระบุไว้ใน Except จะเริ่ม
  ทำงาน
  - ทำงานต่อจาก Try/Except
• หากไม่มีข้อยกเว้น ตัวจัดการจะถูกข้าม

THE ELSE CLAUSE

• Try/Except : อาจรวมถึงประโยคทางเลือกอื่นซึ่งปรากฏต่อท้ายประโยคยกเว้นทั้งหมด
 - สอดคล้องกับ Try/Except
 - ไวยากรณ์คล้ายกับประโยคอื่นในโครงสร้างการตัดสินใจ
 - Else suite : บล็อกของคำสั่งที่ดำเนินการหลังจากคำสั่งในTry เฉพาะในกรณีที่ไม่มีการยกข้อยกเว้น
  - หากยกข้อยกเว้นชุดอื่นจะถูกข้าม

SALE_REPORT4.PY(TRY-EXCEPT-ELSE)

THE FINALLY CLAUSE

• Try/Except : คำสั่งอาจรวมถึงประโยคตัวเลือกสุดท้ายซึ่งปรากฏขึ้นหลังจากทั้งหมดยกเว้นอนุประโยค
 - สอดคล้องกับ Try/Except
 - General format:
   finally:
    statements

 - Finally suite : บล็อกของคำสั่ง
 หลังประโยคสุดท้าย
 - ดำเนินการไม่ว่าจะมีข้อยกเว้นเกิดขึ้น
 หรือไม่
 - จุดประสงค์คือทำการล้างข้อมูลก่อน
 ออก
